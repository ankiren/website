# US-2.1: Create Deck

## Description

As a **user**,
I want to **create a new flashcard deck**,
So that **I can organize my learning by topic or subject**.

---

## Acceptance Criteria

- [x] User can access "New Deck" page from dashboard
- [x] Form requires deck name
- [x] Form accepts optional description
- [x] Deck is created with user's ID as owner
- [x] Successful creation redirects to dashboard
- [x] New deck appears in deck list

---

## Technical Notes

### Implementation
- **API**: `src/app/api/decks/route.ts` (POST)
- **Page**: `src/app/dashboard/decks/new/page.tsx`

### Request/Response
```typescript
// POST /api/decks
Request: { name: string, description?: string }
Response: { id, name, description, userId, createdAt }
```

### Validation
- Name: Required, non-empty string
- Description: Optional string
- User must be authenticated
